
[Git 협업 - 새로운 파일 업로드 순서 (정리)]

1. master 최신 상태 가져오기
----------------------------------
git checkout master
git pull origin master

2. 새로운 브랜치 생성하기 (권장)(필수x)
----------------------------------
git checkout -b 새로운브랜치이름
예) git checkout -b Hyun_v2

3. 작업 파일 추가 또는 수정
----------------------------------
코드를 작성하거나 새로운 파일을 추가/수정

4. 변경사항 스테이지에 추가
----------------------------------
git add .

5. 커밋하기
----------------------------------
git commit -m "작업 내용 설명"

6. 원격에 push
----------------------------------
git push origin 브랜치이름

7. Github에서 Pull Request(PR) 요청
----------------------------------
Github 사이트 접속 → "Compare & pull request" 버튼 클릭 → master로 병합 요청

8. 팀장(또는 권한자)이 Merge
----------------------------------
PR을 검토하고 master에 merge

9. master 최신 상태로 맞추기
----------------------------------
git checkout master
git pull origin master

----------------------------------
※ 반복 작업 흐름
----------------------------------
- 새로운 기능/작업 단위 → 새 브랜치 생성 → 작업 & 커밋 & push → PR → Merge → master pull → 새로운 작업 시작

----------------------------------
※ 참고
----------------------------------
- 기존 브랜치를 계속 쓸 수도 있지만, 보통은 새 기능마다 새로운 브랜치를 만드는 것이 권장됩니다.
- 브랜치 이름은 기능명이나 작업 단계를 알아보기 쉽게 작성하세요. (예: Hyun_fixBug, Hyun_feature1 등)
- master 브랜치에는 직접 push하지 않고 항상 PR(풀 리퀘스트)로 병합합니다.
